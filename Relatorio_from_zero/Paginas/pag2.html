2.3 Etapa 4

  1. Qual opcode é usado para indicar uma solicitação? E para uma resposta?

    2. Opcode: request(1) é utilizado para solicitação.
    Opcode: reply(2) é utilizado para resposta.

  2. Qual é o tamanho do cabeçalho ARP de uma solicitação? E o de uma resposta?

    3. O tamanho do cabeçalho para uma solicitação é o mesmo para a resposta, estes quetem 28 Bytes (224 bits).

  3. Qual valor é indicado em uma solicitação para o endereço MAC de destino desconhecido?

    4. Quando o endereço MAC de destino é desconhecido (Provavelmente estamos falandode um request) apenas é enviado o pacote com o endereço IP de destino e o endereço MAC (destino) é dado por 00:00:00:00:00:00.

  4. Qual é o valor do campo Ethernet Type que indica que o ARP é o protocolo de camada superior?

    5. É indicado por "Type: ARP (0x0806)".

  5. A resposta ARP é transmitida por broadcast (como a solicitação ARP) ou não?

    6. A resposta ARP (reply) não é transmitida por Broadcast uma vez que o endereço dedestino é quem começou a comunicação, logo o pacote contêm o endereço de IP e MAC de ambos, diferente do request que não continha o endereço MAC de destino e por isso necessita do Broadcast.
